# Agent Instructions

This repository uses `AGENTS.md` as the central coordination file for Codex and other coding agents.

## Required Reading

- Before planning or editing this repository, read `docs/project-plan.md`.
- Treat `docs/project-plan.md` as the current working plan and project state summary.
- Update `docs/project-plan.md` when implementation status, priorities, architecture decisions, or working guidelines change.

## Documentation Policy

- Existing docs are partially stale. Verify behavior against code before relying on docs as authoritative.
- When code and docs diverge, either update the docs or call out the mismatch in `docs/project-plan.md`.

## Collaboration Policy

- Keep changes focused on the current task.
- Do not revert unrelated user or agent changes.
- Prefer show-day reliability and clear operational behavior over speculative cloud or packaging work.
